How do Screen Readers Work?

Mon, Dec 2nd, 2024

by Luke Taylor

Note: The demos on the page will play spoken audio. Headphones are recommended.

On the modern web, there are countless resources describing what to do and what not to do in order to make a website accessible. But sometimes these pieces of advice can feel confusing or incomplete without experiencing the problems they solve first-hand.

The following demos aim to help bridge that initial information gap.

This demo will show how different methods of handling images work. To start out with, below is an image with no alt text. Listen for what happens when you focus it.

That wasn't great, was it? With no alt text, the browser falls back on the file name of the image which in this case is just "photo" followed by a long string of numbers and letters. This is both distracting and uninformative.

You actually just missed an image. The previous image has an alt text sent to an empty string. This causes the screen reader to skip over it completely. This is useful if the image is purely decorative like a fancy background or icon. In this case, this is likely fine because the image is a decorative image of some trees.

You just missed another image. But this time it had important content in it about hotel wifi credentials. Without a label there's no way of knowing what it says, and since the alt text is empty, we don't even know we're missing information. In this way, empty alt text should be used, but used carefully.

This is much better, using fully specified alt text. It quickly communicates the essential contents of the image like the username and password, while also ignoring irrelevant info. For example it doesn't matter that the background of the image is purple. This is a good image / alt text pairing.

Rotors: Quicker Navigation

You just focused over a heading. Having properly tagged headings can help experienced screen reader users quickly navigate the page.

This is done through the use of a tool called a rotor. Screen readers typically have rotors for multiple different types of items like headers, links, text boxes, and more. Headings in particular are useful because they typically label larger sections and allow you to skip over large blocks of content. In this demo, you can use a header rotor with H to go to the next header and Shift H to go to the previous one. Jump to the next header now using H.

About the Next Demo

Along the way, you may have noticed different heading levels. These let you know how deep into subsections you are. A larger number means a deeper subsection and typically heading level one is used for a page title. This relies on website authors to setup correctly. It's a big help to screen reader users to know whether they're in a subsection or a main section, so getting these correct can make a site more navigable.
